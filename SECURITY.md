# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in pi-superpowers-plus, please report it via the project's GitHub Issues using the **Bug Report** template:

https://github.com/coctostan/pi-superpowers-plus/issues/new/choose

Please include:
- A description of the vulnerability
- Steps to reproduce (if applicable)
- Any relevant code, configuration, or environment details

Vulnerabilities in transitive dependencies should be reported to the respective upstream package maintainers.

## Scope

This policy covers the `pi-superpowers-plus` npm package — its extensions, agent definitions, and skill files. It does not cover:
- The [`pi` coding agent runtime](https://github.com/badlogic/pi-mono) (report to its maintainers)
- Third-party services or APIs invoked during deployment

## Safe Harbor

We consider security research and responsible disclosure as legitimate contributions.
